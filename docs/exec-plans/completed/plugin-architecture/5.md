# Milestone 5 — Core Models Refactor & Built-in Fetcher Plugins

**Status**: complete

## Scope

This milestone performs two closely related transformations:

1. **Remove `FetchStrategy` enum** from `models.py` and all code that depends on it. `Corpus.fetch_strategy` becomes a plain `str` (the plugin name).
2. **Move fetcher implementations to `_builtins/`** and register them as entry points, replacing the hardcoded dispatch table.

This resolves **Open Question 1 (Built-in vs companion packages)**: built-in plugins ship inside doc-hub in `src/doc_hub/_builtins/`. They are registered via entry points in `pyproject.toml`. This gives install simplicity (one `pip install doc-hub` gets everything) while keeping the architecture clean (built-ins use the same discovery mechanism as external plugins).

from __future__ import annotations

from pathlib import Path
from typing import Any


class LocalDirFetcher:
    """Fetcher plugin for docs already on disk.

    Entry point name: "local_dir"

    Required fetch_config keys:
        path (str): Absolute path to the docs directory.
    """

    async def fetch(
        self,
        corpus_slug: str,
        fetch_config: dict[str, Any],
        output_dir: Path,
    ) -> Path:
        path = Path(fetch_config["path"])
        if not path.is_dir():
            raise FileNotFoundError(f"Local dir not found: {path}")
        return path

### Modify: `src/doc_hub/fetchers.py`

This file is **gutted**. It becomes a thin compatibility shim that:

3. Keeps `fetch()` as the public API but rewrites it to use plugin discovery:

```python
"""Fetcher dispatch for doc-hub.

Routes to the appropriate fetcher plugin via the plugin discovery system.
The hardcoded dispatch table and FetchStrategy enum are gone — any string
name that resolves to a registered Fetcher plugin works.
"""
from __future__ import annotations

from pathlib import Path
from typing import Any

from doc_hub.discovery import get_registry


async def fetch(
    corpus_slug: str,
    fetch_strategy: str,
    fetch_config: dict[str, Any],
    output_dir: Path,
) -> Path:
    """Dispatch to the correct fetcher plugin.

    Args:
        corpus_slug: Corpus identifier (for logging).
        fetch_strategy: Name of the fetcher plugin (e.g. "llms_txt").
        fetch_config: Strategy-specific configuration dict.
        output_dir: Directory where fetched files will be written.

    Returns:
        Path to the directory of fetched files.

    Raises:
        KeyError: If no fetcher plugin with this name is registered.
    """
    registry = get_registry()
    fetcher = registry.get_fetcher(fetch_strategy)
    return await fetcher.fetch(corpus_slug, fetch_config, output_dir)
```

@classmethod
def from_row(cls, row: Any) -> "Corpus":
    # asyncpg Records support dict-like key access (row["key"]) and
    # row.keys() but NOT .get(). Plain dicts support .get().
    # Use hasattr to detect which interface is available.
    if hasattr(row, "get"):
        # Plain dict (e.g., from tests or JSON deserialization)
        embedding_model = row.get("embedding_model") or "gemini-embedding-001"
        parser = row.get("parser") or "markdown"
        embedder = row.get("embedder") or "gemini"
        enabled = row.get("enabled")
        enabled = enabled if enabled is not None else True
        last_indexed_at = row.get("last_indexed_at")
        total_chunks = row.get("total_chunks") or 0
    else:
        # asyncpg Record — use row.keys() to check for column existence.
        # The parser/embedder columns may not exist if reading from
        # a database that was created before M4 was applied (though
        # backward compat is not required, defensive coding is cheap).
        row_keys = set(row.keys())
        embedding_model = row["embedding_model"] if "embedding_model" in row_keys else "gemini-embedding-001"
        parser = row["parser"] if "parser" in row_keys else "markdown"
        embedder = row["embedder"] if "embedder" in row_keys else "gemini"
        enabled = row["enabled"]
        last_indexed_at = row["last_indexed_at"]
        total_chunks = row["total_chunks"] or 0

    return cls(
        slug=row["slug"],
        name=row["name"],
        fetch_strategy=row["fetch_strategy"],  # plain str
        fetch_config=row["fetch_config"],
        embedding_model=embedding_model,  # KEPT until M7
        parser=parser,
        embedder=embedder,
        enabled=enabled,
        last_indexed_at=last_indexed_at,
        total_chunks=total_chunks,

### Modify: `pyproject.toml` — **CRITICAL**

Add entry point registrations so the discovery engine finds the built-in plugins:

```toml
[project.entry-points."doc_hub.fetchers"]
llms_txt = "doc_hub._builtins.fetchers.llms_txt:LlmsTxtFetcher"
local_dir = "doc_hub._builtins.fetchers.local_dir:LocalDirFetcher"
sitemap = "doc_hub._builtins.fetchers.sitemap:SitemapFetcher"
git_repo = "doc_hub._builtins.fetchers.git_repo:GitRepoFetcher"
```

Without these entry points, `get_registry().list_fetchers()` will return an empty list and the pipeline will fail with `KeyError: 'llms_txt'`.

**Note on entry points:** Entry points are read from installed package metadata, not from `pyproject.toml` on disk. At runtime (after `pip install` or `uv sync`), `importlib.metadata.entry_points(group="doc_hub.fetchers")` discovers them automatically. **For tests, mock `importlib.metadata.entry_points()`** to return `EntryPoint` objects pointing at the builtin classes — this avoids requiring a package reinstall during verification.

**Note:** The parser and embedder entry points are added in Milestones 6 and 7 respectively.

## Design Decisions

1. **Built-in plugins ship inside doc-hub**: The `_builtins/` subpackage prefix with underscore signals these are internal. They use the same plugin mechanism as external plugins.

2. **Manifest helpers stay with llms_txt**: These are specific to the llms_txt incremental sync pattern. Other fetchers (sitemap, git_repo) will have their own caching mechanisms.

3. **`fetch()` signature change**: The public `fetch()` function now takes individual arguments instead of a `Corpus` object. This keeps the fetcher interface clean and avoids coupling fetcher dispatch to the `Corpus` model.
