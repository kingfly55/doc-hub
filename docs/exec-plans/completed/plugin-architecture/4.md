# Milestone 4 — Database Config Cleanup & Schema Changes

**Status**: complete

## Scope

Clean up the database layer to remove hardcoded assumptions and prepare for pluggable embedders:

1. **Remove the `CHECK` constraint** on `fetch_strategy` — the value is now a freeform string (plugin name), not an enum.
2. **Clean up connection defaults**: standard port `5432` (not `5433`), default database name `doc_hub` (not `postgres`), no default password. **Breaking change:** The current defaults are `PGPORT=5433`, `PGDATABASE=postgres`, `PGPASSWORD=pydantic-docs`. After this change, `PGPASSWORD` has no default and must be set explicitly. Users of the existing Docker setup (`-p 5433:5432 -e POSTGRES_PASSWORD=pydantic-docs`) must update their env vars or use `DOC_HUB_DATABASE_URL`.
3. **Add `DOC_HUB_DATABASE_URL` env var**: single connection string override. Takes precedence over individual `PG*` env vars.
4. **Handle vector dimensions**: resolve Open Question 3 — all corpora in a single deployment must use the same embedding dimensions. The dimension is configurable at schema creation time via `DOC_HUB_VECTOR_DIM` env var (default: 768).
5. **Remove legacy migration**: backward compatibility with `pydantic-ai-docs` is not required (Constraint 4). Remove `migrate_from_legacy()` entirely.

### Resolution of Open Question 3 (Vector Dimensions)

**Decision: Single dimension per deployment, configurable at schema creation time.**

Rationale:
- pgvector/VectorChord `vector(N)` is a fixed-width column type. Mixing dimensions in one table requires either zero-padding (wasteful), ALTER TABLE (dangerous), or separate tables (complex).
- The simplest approach: `DOC_HUB_VECTOR_DIM` env var sets the dimension when `ensure_schema()` creates the table. Default is 768 (Gemini). If a user wants OpenAI 1536-dim, they set `DOC_HUB_VECTOR_DIM=1536` before first run.
- The embed pipeline validates that the embedder's `dimensions` property matches the configured dimension and fails fast if they mismatch (done in Milestone 7).
- This is a deployment-time decision, not a per-corpus decision. Documented clearly.

**1. Rewrite `_build_dsn()`:**

```python
def _build_dsn(dsn: str | None = None) -> str:
    """Return DSN from argument or environment variables.

    Resolution order:
    1. Explicit dsn argument
    2. DOC_HUB_DATABASE_URL env var (full connection string)
    3. Individual PG* env vars with safe defaults

    Env vars:
      DOC_HUB_DATABASE_URL — full PostgreSQL connection string
      PGHOST     (default: localhost)
      PGPORT     (default: 5432)  # standard PostgreSQL port
      PGDATABASE (default: doc_hub)
      PGUSER     (default: postgres)
      PGPASSWORD (NO default — must be set explicitly)
    """
    if dsn:
        return dsn

    env_url = os.getenv("DOC_HUB_DATABASE_URL")
    if env_url:
        return env_url

    host = os.getenv("PGHOST", "localhost")
    port = os.getenv("PGPORT", "5432")
    dbname = os.getenv("PGDATABASE", "doc_hub")
    user = os.getenv("PGUSER", "postgres")
    password = os.getenv("PGPASSWORD")
    if not password:
        raise RuntimeError(
            "PGPASSWORD environment variable not set. "
            "Set it directly or use DOC_HUB_DATABASE_URL for the full connection string. "
            "Note: PGPASSWORD no longer has a default value (previously defaulted to 'pydantic-docs')."
        )
    # URL-encode user and password to handle special characters (@, /, %, etc.)
    from urllib.parse import quote_plus
    return f"postgresql://{quote_plus(user)}:{quote_plus(password)}@{host}:{port}/{dbname}"
```

**3. Parameterize vector dimension — replace `_CHUNKS_DDL` constant with `_chunks_ddl()` function:**

The current `_CHUNKS_DDL` is a module-level string constant with `vector(768)` hardcoded. Replace it with a function that reads the dimension from the environment. All callers that reference `_CHUNKS_DDL` (currently just `ensure_schema()`) must change to call `_chunks_ddl()` instead.

```python
def get_vector_dim() -> int:
    """Return the configured vector dimension for this deployment.

    Reads DOC_HUB_VECTOR_DIM env var (default: 768).
    Used by the embed pipeline to validate embedder compatibility.

    Raises:
        ValueError: If DOC_HUB_VECTOR_DIM is set but not a positive integer.
    """
    raw = os.getenv("DOC_HUB_VECTOR_DIM", "768")
    try:
        dim = int(raw)
    except ValueError:
        raise ValueError(
            f"DOC_HUB_VECTOR_DIM must be a positive integer, got {raw!r}"
        ) from None
    if dim <= 0:
        raise ValueError(
            f"DOC_HUB_VECTOR_DIM must be a positive integer, got {dim}"
        )
    return dim


def _chunks_ddl() -> str:
    """Generate doc_chunks DDL with the configured vector dimension."""
    dim = get_vector_dim()
    return f"""
CREATE TABLE IF NOT EXISTS doc_chunks (
    id           serial PRIMARY KEY,
    corpus_id    text NOT NULL REFERENCES doc_corpora(slug) ON DELETE CASCADE,
    content_hash text NOT NULL,
    heading      text NOT NULL,
    content      text NOT NULL,
    tsv          tsvector GENERATED ALWAYS AS (
                     setweight(to_tsvector('english', heading), 'A') ||
                     setweight(to_tsvector('english', content), 'B')
                 ) STORED,
    embedding    vector({dim}) NOT NULL,
    source_file  text NOT NULL,
    source_url   text NOT NULL,
    section_path text NOT NULL,
    heading_level smallint NOT NULL,
    start_line   int NOT NULL DEFAULT 0,
    end_line     int NOT NULL DEFAULT 0,
    char_count   int NOT NULL,
    category     text NOT NULL,
    UNIQUE (corpus_id, content_hash)
)
"""
```

**5. Simplify `ensure_schema()`:**

Remove ALL legacy code: the `migrate_from_legacy()` call, the legacy table detection logic, AND the `start_line`/`end_line` column migration check (lines 245–260 of current `db.py`). Since backward compatibility with existing data is not required (Constraint 4), `ensure_schema` should be a clean "create if not exists" with no migration logic.

```python
async def ensure_schema(pool: asyncpg.Pool) -> None:
    """Create all doc-hub tables and indexes if they don't exist.

    Uses the VectorChord extension (vchord).
    This function is idempotent — safe to call on every startup.

    IMPORTANT: After creating/verifying the doc_chunks table, this
    function checks that the existing vector column dimension matches
    DOC_HUB_VECTOR_DIM. If they differ, it raises RuntimeError with
    instructions — because CREATE TABLE IF NOT EXISTS silently
    preserves the old schema, the mismatch would otherwise cause
    cryptic INSERT failures later.
    """
    async with pool.acquire() as conn:
        await conn.execute("CREATE EXTENSION IF NOT EXISTS vchord CASCADE")
        log.debug("VectorChord extension ensured.")

        await conn.execute(_CORPORA_DDL)
        log.debug("doc_corpora table ensured.")

        await conn.execute(_chunks_ddl())
        log.debug("doc_chunks table ensured.")

        # Validate existing vector dimension matches configured dimension.
        # CREATE TABLE IF NOT EXISTS preserves the old schema, so if
        # DOC_HUB_VECTOR_DIM changed after initial creation, the table
        # still has the old dimension. Detect this and fail fast.
        existing_dim = await conn.fetchval(
            """
            SELECT atttypmod
            FROM pg_attribute
            WHERE attrelid = 'doc_chunks'::regclass
              AND attname = 'embedding'
              AND NOT attisdropped
            """
        )
        # atttypmod for vector(N) is N. If the column doesn't exist
        # or has no typmod, existing_dim will be None or -1.
        configured_dim = get_vector_dim()
        if existing_dim is not None and existing_dim > 0 and existing_dim != configured_dim:
            raise RuntimeError(
                f"Existing doc_chunks table has vector({existing_dim}) but "
                f"DOC_HUB_VECTOR_DIM={configured_dim}. To fix this, either:\n"
                f"  1. Set DOC_HUB_VECTOR_DIM={existing_dim} to match the existing table, or\n"
                f"  2. DROP TABLE doc_chunks and let doc-hub recreate it with the new dimension.\n"
                f"     (This will delete all indexed data — re-index all corpora after.)"
            )

        await conn.execute(_META_DDL)
        log.debug("doc_index_meta table ensured.")

        for stmt in _INDEXES_DDL.strip().split("\n\n"):
            stmt = stmt.strip()
            if stmt:
                await conn.execute(stmt)
        log.debug("Indexes ensured.")

    log.info("doc-hub schema verified / created.")
```

**6. Update `upsert_corpus()`:**

The current SQL references `embedding_model` and uses `corpus.fetch_strategy.value` (enum). The SQL must change to use `parser`/`embedder` columns instead of `embedding_model`:

```python
async def upsert_corpus(pool: asyncpg.Pool, corpus) -> None:
    """Insert or update a corpus row."""
    await pool.execute(
        """
        INSERT INTO doc_corpora (slug, name, fetch_strategy, fetch_config,
                                 parser, embedder, enabled)
        VALUES ($1, $2, $3, $4::jsonb, $5, $6, $7)
        ON CONFLICT (slug) DO UPDATE SET
            name            = EXCLUDED.name,
            fetch_strategy  = EXCLUDED.fetch_strategy,
            fetch_config    = EXCLUDED.fetch_config,
            parser          = EXCLUDED.parser,
            embedder        = EXCLUDED.embedder,
            enabled         = EXCLUDED.enabled
        """,
        corpus.slug,
        corpus.name,
        corpus.fetch_strategy.value,  # still .value — FetchStrategy enum exists until M5
        json.dumps(corpus.fetch_config),
        corpus.parser,
        corpus.embedder,
        corpus.enabled,
    )
```

Note: `corpus.fetch_strategy` is still a `FetchStrategy` (StrEnum) at this point — the enum is not removed until M5. Since `StrEnum` is a `str` subclass, both `corpus.fetch_strategy` and `corpus.fetch_strategy.value` produce the same string. Using `.value` is explicit and consistent with the current code. M5 removes `.value` when `FetchStrategy` is deleted.
