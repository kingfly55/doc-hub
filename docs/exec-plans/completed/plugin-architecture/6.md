# Milestone 6 — Parser Pluggability

**Status**: complete

## Scope

Make the parser stage pluggable by splitting `parse.py` into:

1. **Core pipeline logic** (stays in `parse.py`): chunk size optimization (merge tiny, split mega), deduplication by content hash, category derivation, `embedding_input()`, JSONL I/O. This code is universal and runs regardless of which parser plugin produced the raw chunks.

2. **Built-in markdown parser** (moves to `_builtins/parsers/markdown.py`): heading-based splitting of `.md` files, section path construction, manifest loading for URL mapping. This is the default parser plugin.

The `Chunk` dataclass stays in `parse.py` — it's a core data type, not parser-specific.

### Parser Boundary (Open Question 2 Resolution)

**What the parser does:**
- Reads files from an input directory
- Splits them into raw `Chunk` objects (one per logical section)
- Sets: `source_file`, `source_url`, `section_path`, `heading`, `heading_level`, `content`, `start_line`, `end_line`, `char_count`, `content_hash`

**What the parser does NOT do (core pipeline handles these):**
- Merge tiny chunks (`_merge_tiny_chunks`)
- Split mega chunks (`_split_mega_chunks`)
- Deduplicate by content hash
- Derive category (`derive_category`)
- Write JSONL output

This boundary means a parser for RST, AsciiDoc, or HTML only needs to produce `Chunk` objects — the size optimization and dedup logic is reused automatically.

```python
class MarkdownParser:
    """Parser plugin for markdown files with heading-based splitting.

    Reads .md files from an input directory, splits each by ATX headings
    (# through ######), and returns a list of raw Chunk objects.

    Files starting with '_' are skipped (e.g. _llms.txt manifest).
    When a manifest.json is present, only files listed in it are parsed
    (prevents indexing orphaned files after upstream removal).
    """

    def parse(
        self,
        input_dir: Path,
        *,
        corpus_slug: str,
        base_url: str,
    ) -> list[Chunk]:
        """Parse markdown files into raw chunks.

        Args:
            input_dir: Directory containing .md files.
            corpus_slug: Corpus identifier (for logging).
            base_url: Base URL for source_url reconstruction.

        Returns:
            List of raw Chunk objects (before size optimization).
        """
        url_map = self._load_manifest(input_dir)

        if url_map:
            md_files = sorted(
                input_dir / fn for fn in url_map
                if (input_dir / fn).exists()
            )
        else:
            md_files = sorted(input_dir.glob("*.md"))

        md_files = [f for f in md_files if not f.name.startswith("_")]
        log.info("[%s] Parsing %d markdown files", corpus_slug, len(md_files))

        all_chunks: list[Chunk] = []
        for md_file in md_files:
            text = md_file.read_text(errors="replace")
            source_url = url_map.get(md_file.name, "")
            chunks = self._split_into_chunks(text, md_file.name, source_url)
            all_chunks.extend(chunks)

            "[%s] Raw chunks from heading split: %d",
            corpus_slug,
            len(all_chunks),
```

**`_is_fence_marker()` duplication:** This function is needed by both `parse.py` (`_find_safe_split`) and `MarkdownParser` (`_parse_headings`). **Decision: `MarkdownParser` defines its own `_is_fence_marker` as a `@staticmethod`.** Rationale: (a) importing private functions (`_`-prefixed) from another module is an anti-pattern that couples the parser to `parse.py` internals, (b) the function is trivial (3 lines: `stripped = line.strip(); return stripped.startswith("```") or stripped.startswith("~~~")`), and (c) this keeps the `MarkdownParser` import surface minimal (`from doc_hub.parse import Chunk` only). If a future parser also needs fence detection, extract to a shared `_utils.py` at that time (YAGNI).

**`_make_chunk()` clarification:** There will be TWO `_make_chunk` functions after the split:
1. `parse.py:_make_chunk()` — calls `derive_category(source_file)` to set `category`. Used by `_merge_tiny_chunks` and `_split_mega_chunks` when creating new chunks during size optimization.
2. `MarkdownParser._make_chunk()` — sets `category=""` (empty string). Used by the parser when creating raw chunks from heading splits. The core pipeline fills in categories after the parser returns.

```python
def parse_docs(
    corpus_slug: str,
    raw_path: Path,
    parser_name: str = "markdown",
    base_url: str = "",
) -> list[Chunk]:
    """Parse raw files into optimized, deduplicated chunks.

    1. Resolve the parser plugin by name
    2. Call parser.parse() to get raw chunks
    3. Derive categories
    4. Merge tiny chunks (< 500 chars)
    5. Split mega chunks (> 2500 chars)
    6. Deduplicate by content hash
    7. Write chunks.jsonl

    Args:
        corpus_slug: Corpus slug (for paths and logging).
        raw_path: Directory containing raw files from the fetch stage.
        parser_name: Name of the parser plugin to use (default "markdown").
        base_url: Base URL for reconstructing source URLs from filenames.
            Passed through to the parser plugin. The caller (pipeline.py)
            reads this from corpus.fetch_config.get("base_url", "").

    Returns:
        List of optimized, deduplicated Chunk objects.
    """
    from doc_hub.discovery import get_registry

    registry = get_registry()
    parser = registry.get_parser(parser_name)

    # Step 1: Parser produces raw chunks
    all_chunks = parser.parse(raw_path, corpus_slug=corpus_slug, base_url=base_url)

    # Step 2: Derive categories (parser leaves category empty)
    for chunk in all_chunks:
        if not chunk.category:
            chunk.category = derive_category(chunk.source_file)

    log.info("[%s] Initial chunks: %d", corpus_slug, len(all_chunks))

    # Step 3-6: Size optimization, dedup (unchanged from current code)
    all_chunks = _merge_tiny_chunks(all_chunks, min_chars=500)
    all_chunks = _split_mega_chunks(all_chunks, max_chars=2500, target=1000)

    # Dedup
    seen_hashes: set[str] = set()
    deduped: list[Chunk] = []
    for chunk in all_chunks:
        if chunk.content_hash not in seen_hashes:
            seen_hashes.add(chunk.content_hash)
            deduped.append(chunk)
    all_chunks = deduped

    _warn_large_chunks(all_chunks)

    # Step 7: Write output
    from doc_hub.paths import chunks_dir
    output_dir = chunks_dir(corpus_slug)
    output_dir.mkdir(parents=True, exist_ok=True)
    write_chunks_jsonl(all_chunks, output_dir / "chunks.jsonl")

    log.info("[%s] Total: %d chunks", corpus_slug, len(all_chunks))
    return all_chunks
```

**Note on `Chunk` mutability:** The `Chunk` dataclass is not frozen, so `chunk.category = derive_category(...)` works via direct attribute assignment. The current code already creates new chunks via `_make_chunk` in merge/split, so mutation patterns are established.

```python
async def run_parse(corpus: Corpus) -> list:
    from doc_hub.parse import parse_docs
    log.info("[%s] === STEP 2: Parse (parser=%s) ===", corpus.slug, corpus.parser)
    raw_path = raw_dir(corpus)
    base_url = corpus.fetch_config.get("base_url", "")
    chunks = parse_docs(
        corpus.slug, raw_path,
        parser_name=corpus.parser,
        base_url=base_url,
    )
    log.info("[%s] Parse complete → %d chunks", corpus.slug, len(chunks))
    return chunks
```

**Note on `base_url` propagation:** The `base_url` is currently used by the markdown parser to reconstruct `source_url` from filenames when no manifest is present. For the built-in `MarkdownParser`, the manifest (written by the fetcher) provides the URL mapping, so `base_url` is a fallback. Custom parsers may use `base_url` differently or ignore it entirely. The protocol makes it a required keyword argument so all parsers receive it — they can choose not to use it.

**Note on `parse_docs()` calling `chunks_dir(corpus_slug)`:** After M3, `chunks_dir()` accepts a plain string slug. The `parse_docs()` function uses `chunks_dir(corpus_slug)` to determine the output path for `chunks.jsonl`. **This is a hard dependency on M3 being complete.** If M3 is not done, this call will fail because the current `chunks_dir(corpus)` expects a `Corpus` object, not a string. The implementation agent must ensure M3 is applied before M6.

**Note on `parse_docs()` removing the `Corpus` import:** After this rewrite, `parse.py` no longer imports `Corpus` from `models.py` at the top level (the current `from doc_hub.models import Corpus` on line 23 of parse.py). Remove this import. The `chunks_dir` import from `paths.py` stays — it's used in `parse_docs()` to write output. The `paths.py` import of `Corpus` was moved to a `TYPE_CHECKING` guard in M3, so there's no circular dependency risk.
