## Detailed Specification

### 1. `browse_corpus_tool` — MCP tool handler

```python
@server.tool()
async def browse_corpus_tool(
    corpus: str,
    ctx: Context,
    path: str | None = None,
    depth: int | None = None,
) -> list[dict]:
    """Browse the document hierarchy of a corpus.

    Returns the document tree structure, showing how documentation is
    organized into sections and pages. Use this to understand the
    corpus structure before searching or reading specific documents.
    """
```

Required behavior:
- Delegate directly to `_browse_corpus_impl()` with the shared lifespan pool.
- Expose `path` as a subtree prefix filter and `depth` as a relative maximum depth.
- Return the document tree data unchanged so the CLI can reuse the same shape.

### 2. `_browse_corpus_impl` — testable core logic

```python
async def _browse_corpus_impl(
    *,
    corpus: str,
    path: str | None,
    depth: int | None,
    pool: asyncpg.Pool,
) -> list[dict]:
    """Core browse logic, callable directly in tests without MCP framework."""
```

Required behavior:
- Import `get_document_tree` locally to avoid circular imports.
- Delegate to `get_document_tree(pool, corpus, path=path, max_depth=depth)` and return the result unchanged.
- Do not post-process the tree or rename fields.

### 3. `get_document_tool` — MCP tool handler

```python
@server.tool()
async def get_document_tool(
    corpus: str,
    doc_path: str,
    ctx: Context,
    section: str | None = None,
    force: bool = False,
) -> dict:
    """Read a document's content from the corpus."""
```

Required behavior:
- Delegate directly to `_get_document_impl()` with the shared lifespan pool.
- When a section is requested, return only that section's chunks in full-content mode rather than an outline.
- Support `force=True` to bypass the large-document outline threshold.

### 4. `_get_document_impl` — testable core logic

```python
LARGE_DOC_THRESHOLD = 20_000  # chars

async def _get_document_impl(
    *,
    corpus: str,
    doc_path: str,
    section: str | None,
    force: bool,
    pool: asyncpg.Pool,
) -> dict:
    """Core get-document logic, callable directly in tests without MCP framework."""
```

Required behavior:
- Import `get_document_chunks` locally to avoid circular imports.
- Call `get_document_chunks(pool, corpus, doc_path, section=section)`.
- If no chunks are returned, return `{"error": f"Document '{doc_path}' not found in corpus '{corpus}'"}`.
- Compute `total_chars` and `section_count` from the returned chunks.
- Derive `title` from the first chunk whose `heading_level == 1`; otherwise fall back to `doc_path`.
- `source_url` should come from the first returned chunk.
- If `total_chars > LARGE_DOC_THRESHOLD`, `force` is false, and `section is None`, return `mode="outline"` with a sections list built from the already-loaded chunks and a hint explaining how to read a section or force full output.
- Otherwise return `mode="full"` with the chunks concatenated in `\n\n` order.

Required response shapes:
- Full mode: `mode`, `doc_path`, `title`, `content`, `source_url`, `total_chars`, `section_count`
- Outline mode: `mode`, `doc_path`, `title`, `source_url`, `total_chars`, `section_count`, `sections`, `hint`
