## Detailed Specification

### 1. `upsert_documents(pool, corpus_slug, nodes) -> dict[str, int]`

Persist the tree in two passes so parent rows always exist before child `parent_id` updates are applied.

- **Pass 1**: insert/update every node with `parent_id = NULL`.
- **Pass 2**: update `parent_id` for every node, including roots. Roots and nodes whose parent is unresolved must end up with `parent_id = NULL` so stale parent pointers cannot survive a rebuild.

Return `{doc_path: id}` for every persisted node.

```python
async def upsert_documents(
    pool: asyncpg.Pool,
    corpus_slug: str,
    nodes: list[DocumentNode],
) -> dict[str, int]:
    """Upsert DocumentNode list into doc_documents. Returns {doc_path: id}."""
```

Required behavior:
- Upsert on `(corpus_id, doc_path)`.
- Persist `title`, `source_url`, `source_file`, `depth`, `sort_order`, `is_group`, `total_chars`, and `section_count`.
- Do not try to set `parent_id` in the insert statement; set it only in pass 2.
- Use the returned `id` from `RETURNING` to build the `path_to_id` map.
- In pass 2, update every row in `nodes`, setting `parent_id` from the parent node's id when `node.parent_path` is present in `path_to_id`, otherwise `NULL`.

This makes rebuilds idempotent when tree shape changes.

### 2. `link_chunks_to_documents(pool, corpus_slug, path_to_id) -> int`

Update `doc_chunks.document_id` by linking chunks to the concrete document rows that own them.

The reliable match key is `doc_documents.source_file`, not the namespaced `doc_path`, because concrete document nodes preserve the original filename while group nodes do not. `path_to_id` is still accepted so the call signature stays aligned with `upsert_documents()`, but the authoritative lookup source is the persisted `doc_documents` rows.

Implementation requirements:
- Query `doc_documents` for all rows in the corpus where `source_file <> ''`.
- Build an exact `source_file -> doc_id` map from those rows.
- For each distinct `doc_chunks.source_file` in the corpus, update `doc_chunks.document_id` to the matching document id.
- If a chunk source file is already linked to the right document id, updating it again is fine; the operation must be idempotent.
- If a chunk source file cannot be matched to a document row, skip it rather than guessing.

The function should return the number of updated chunk rows.

### 3. `delete_stale_documents(pool, corpus_slug, current_paths) -> int`

Remove `doc_documents` rows whose `doc_path` is not in `current_paths`.

```python
async def delete_stale_documents(
    pool: asyncpg.Pool,
    corpus_slug: str,
    current_paths: set[str],
) -> int:
    """Delete doc_documents not in current_paths. Returns deleted count."""
```

Required SQL behavior:
- Use `NOT (doc_path = ANY($2::text[]))` rather than `!= ALL(...)` so the empty-set case deletes everything cleanly.
- Delete only rows in the target corpus.
- Deleting a document row must rely on the schema FK to null out any linked `doc_chunks.document_id` values.

### 4. `get_document_tree(pool, corpus_slug, path=None, max_depth=None) -> list[dict]`

Query the document tree, optionally scoped to a subtree rooted at `path` and limited to `max_depth` levels.

```python
async def get_document_tree(
    pool: asyncpg.Pool,
    corpus_slug: str,
    path: str | None = None,
    max_depth: int | None = None,
) -> list[dict]:
    """Return document tree as list of dicts.

    Falls back to synthetic flat list from doc_chunks when no doc_documents exist.
    """
```

Required behavior:
- If the corpus has no `doc_documents` rows at all, call `_synthetic_tree_fallback()` and return a flat list built from `doc_chunks`.
- If a `path` is provided, return that node plus all descendants using a prefix filter (`doc_path = $path OR doc_path LIKE $path || '/%'`).
- If `max_depth` is provided, interpret it relative to the requested subtree root, not absolute database depth.
- Order rows by `sort_order` only.
- Return dictionaries with at least: `doc_path`, `title`, `source_url`, `depth`, `is_group`, `total_chars`, `section_count`, `children_count`.
- Compute `children_count` from the fetched result set, not with a separate query per node.

The synthetic fallback must stay flat and deterministic; it does not try to invent group nodes that were never persisted.

### 5. `_synthetic_tree_fallback(pool, corpus_slug) -> list[dict]`

When no `doc_documents` rows exist, build a flat list from `doc_chunks`.

Required behavior:
- Group by `source_file`.
- Compute `total_chars` as `SUM(char_count)`.
- Compute `section_count` as `COUNT(*)`.
- Use the first available `source_url` for the file.
- Use `_doc_path_from_source_file(source_file)` for `doc_path`.
- Humanize the last path segment for the title.
- Return `is_group=False`, `depth=0`, and `children_count=0` for every row.
- Order the fallback list by `source_file` for deterministic output.

### 6. `get_document_chunks(pool, corpus_slug, doc_path, section=None) -> list[dict]`

Return all chunks for a document, ordered by `start_line`.

Required behavior:
- Prefer the persisted tree when available: resolve the `doc_documents` row for `corpus_id + doc_path`, then fetch chunks by `document_id`.
- If the document row does not exist or the corpus has not been tree-built yet, fall back to `source_file` matching using the base doc path.
- For namespaced doc paths from section-based trees, strip the leading `_section/<slug>/` namespace before deriving the fallback `source_file`.
- When `section` is provided, filter on the heading hierarchy prefix using `section_path = $section OR section_path LIKE $section || ' > %'` so only the chosen section and its descendants are returned.
- Return dict rows with the same chunk fields needed by the read tools: `id`, `heading`, `section_path`, `content`, `source_url`, `heading_level`, `start_line`, `end_line`, `char_count`, `category`.

Implementation note:
- A two-step lookup is clearer than a single OR-heavy SQL statement: resolve the document row first, then choose the document-id path or the source-file fallback path.

### 7. `get_document_sections(pool, corpus_slug, doc_path) -> list[dict]`

Return within-document heading outline with char counts.

```python
async def get_document_sections(
    pool: asyncpg.Pool,
    corpus_slug: str,
    doc_path: str,
) -> list[dict]:
    """Return section outline for a document (heading, level, char_count)."""
```

Required behavior:
- Reuse `get_document_chunks()` so the section outline stays consistent with the read path.
- Return `{heading, heading_level, section_path, char_count}` per chunk.

## Success Criteria

1. `upsert_documents()` inserts all nodes and correctly sets `parent_id` in the second pass.
2. `upsert_documents()` clears stale `parent_id` values when a node becomes a root or its parent is missing.
3. `link_chunks_to_documents()` updates `document_id` on `doc_chunks` rows by matching exact `source_file` values to persisted document rows.
4. `delete_stale_documents()` removes only `doc_documents` rows not in `current_paths`, including the empty-set case.
5. `get_document_tree()` returns the tree scoped by optional `path` and `max_depth`, where `max_depth` is relative to the subtree root.
6. `get_document_tree()` falls back to `_synthetic_tree_fallback()` when no `doc_documents` rows exist.
7. `get_document_chunks()` returns ordered chunks for a document, with section filter support, and handles both persisted-tree and pre-tree corpora.
8. `get_document_sections()` returns a heading outline with char counts.
9. All functions are importable from `doc_hub.documents`.
