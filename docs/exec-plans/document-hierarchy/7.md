### 2. `_render_tree(nodes) -> str`

Format the document tree as an indented text tree view. Use simple space-based indentation (not Unicode box-drawing characters) for terminal compatibility.

```python
def _render_tree(nodes: list[dict]) -> str:
    """Render a flat list of document nodes as an indented tree."""
    if not nodes:
        return "(no documents)"

    lines: list[str] = []
    for node in nodes:
        indent = "    " * node["depth"]

        if node["is_group"]:
            detail = "[group]"
        else:
            chars = f"{node['total_chars']:,} chars"
            sects = f"{node['section_count']} section{'s' if node['section_count'] != 1 else ''}"
            detail = f"{chars}  {sects}"

        title = node["title"]
        lines.append(f"{indent}{title} {detail}")

    return "\n".join(lines)
```

Required behavior:
- Preserve the ordering returned by `get_document_tree()`; do not sort or otherwise mutate it.
- Indent by exactly 4 spaces per depth level.
- Render group nodes with `[group]` and content nodes with `N chars` / `N section(s)` metadata.
- Keep the output readable without relying on fixed-width alignment that could mislead tests.

### 4. `read_main()` — entry point for `doc-hub-read`

```python
LARGE_DOC_THRESHOLD = 20_000  # chars — matches MCP tool threshold
```

Required behavior:
- Reuse the same threshold as the MCP tool.
- Load chunks via `get_document_chunks(pool, args.corpus, args.doc_path, section=args.section)`.
- If no chunks are found, print a not-found message and exit successfully.
- Compute the title the same way as the MCP tool: first H1 chunk, otherwise `doc_path`.
- For large docs (`total_chars > LARGE_DOC_THRESHOLD`) with no `--force` and no `--section`, print an outline instead of full content.
- When `--section` or `--force` is used, print full content.
- `--json` should emit the same shape as the MCP tool (`mode`, `doc_path`, `title`, `content` or `sections`, etc.).

`_render_outline(sections)` should render section headings with indentation based on `heading_level` and include char counts.
