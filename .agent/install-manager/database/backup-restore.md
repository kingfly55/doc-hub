# Backup and Restore Notes

This file is intentionally conservative.

## Before risky repair actions

Before any destructive DB action, confirm:
- which database you are connected to
- whether the install is local-only or shared
- whether data can be regenerated by reindexing
- whether operator approval is needed

## Safe assumptions

- Most doc-hub indexed data can be regenerated from source docs plus corpus registrations
- That does not mean destructive resets are free; they still remove local state and take time to rebuild

## Recommended posture

- prefer targeted repairs first
- only use drop/recreate workflows when the root cause requires it
- if using a managed provider, use the provider’s native backup/restore tools
